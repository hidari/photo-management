# イベント作成ガイド

## 概要

新しい撮影イベントを作成するコマンドと、モデルを追加するためのコマンドでイベント情報を構成します。

## イベント初期化

### コマンド

```bash
deno task init
```

### 実行内容

対話形式でイベント情報を入力すると、以下が自動で実行されます：

1. イベント情報の入力
   - 開催日（例: 20240315）
   - イベント名（例: 春の撮影会）
   - 参加モデル情報

2. ディレクトリ構造の自動作成
   ```
   20240315_春の撮影会/
   ├── 
   ├── distribution.config.toml
   ├── 20240315_春の撮影会_{撮影者}_モデルAさん/
   │   └── README.txt
   ├── 20240315_春の撮影会_{撮影者}_モデルBさん/
   │   └── README.txt
   └── ...
   ```

3. README.txtの生成
   - 写真の利用規約
   - テンプレートから自動生成

4. distribution.config.tomlの保存
   - イベント設定の永続化
   - モデル情報の保存
   - 後から `deno task add` で追加可能

## モデルの追加

イベント作成後、または既存のイベントにモデルを追加する場合に使用します。

### コマンド

```bash
# TOML編集後、差分を同期
deno task add

# 対話的にモデルを追加
deno task add --dialog
```

### 使い分け

#### `deno task add`（TOML編集方式）

`distribution.config.toml` を直接編集してからコマンドを実行：

1. `distribution.config.toml` を開く
2. `[[events.models]]` セクションにモデル情報を追加
3. `deno task add` を実行してディレクトリを同期

メリット: 複数のモデルを一度に追加できる、コピー＆ペーストで効率的

#### `deno task add --dialog`（対話方式）

対話形式でモデル情報を入力：

```bash
$ deno task add --dialog
```

メリット: 手軽に1人ずつ追加できる

## ワークフロー例

### 新規イベント作成の流れ

1. イベントを作成
   ```bash
   deno task init
   ```

2. 写真を配置
   現像済みのファイルを各モデルのディレクトリに出力/コピー

3. 配布準備へ進む

   次は[写真配布ガイド](./写真配布ガイド.md)を参照してください。

## 次のステップ

イベント作成とモデル追加が完了したら、次は[写真配布ガイド](./写真配布ガイド.md)を参照して写真をアップロードして配布してください。
